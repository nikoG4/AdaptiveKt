# Pages Link Validation

## Overview
To prevent 404 errors from unlinked static site demos during the GitHub Pages deployment cycle, we introduced automated static route checks. Because the Compose WebAssembly application is generated as index.html envelopes linking out to binary payloads, ensuring directory topology is critical.

## Check Script
`scripts/check-pages-links.ps1` (and `.sh`) verifies the presence of the mapped index files inside `site-dist/` generated by the preparation script. 

It explicitly validates the existence of:
- `index.html` (Docs Site)
- `demo/app/index.html` (Admin Demo)
- `examples/ecommerce/index.html` (Ecommerce Demo)
- `examples/ai-workspace/index.html` (AI Workspace Demo)

## Result
Running the script yields a pass across all static paths, confirming the AI Workspace has successfully merged into the deployment staging payload.
