# Code of Conduct

## Our Pledge

AdaptiveKt is a small open-source project, and its community should be useful, respectful, and welcoming. Contributors, maintainers, and users are expected to participate with consideration for different backgrounds, experience levels, languages, and time zones.

## Expected Behavior

- Use welcoming and inclusive language.
- Be respectful of different viewpoints and technical tradeoffs.
- Give and receive feedback constructively.
- Focus discussion on what helps the project and its users.
- Assume good intent, while still being clear about impact.

## Unacceptable Behavior

- Harassment, intimidation, insults, or discriminatory language.
- Personal attacks, trolling, or repeated bad-faith arguments.
- Publishing private information without permission.
- Sexualized language or imagery in project spaces.
- Any conduct that would reasonably make participation unsafe or hostile.

## Scope

This Code of Conduct applies in AdaptiveKt project spaces, including GitHub issues, pull requests, discussions, documentation, examples, and any community channel used for project work. It also applies when a person is clearly representing AdaptiveKt in public.

## Enforcement

Project maintainers may remove comments, close issues, reject contributions, or temporarily/permanently block participation when behavior violates this Code of Conduct. Maintainers should prefer clear, proportionate, and documented responses.

## Reporting

If you experience or observe unacceptable behavior, contact the maintainer privately at `nikoovelarrealg4@gmail.com`. Do not open a public issue for sensitive conduct reports.

This project also takes inspiration from the Kotlin community expectation of respectful, safe, and welcoming participation.
